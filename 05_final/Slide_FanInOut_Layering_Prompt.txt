SLIDE 4 — 5W1H ANALYTICAL QUESTIONS (Rubric Sec 2: 10 pts)
================================================================

WHO — บัญชีประเภทไหนเป็นบัญชีม้า?
  → ใช้คอลัมน์: Account_Age_Days, Layer
  → สมมติฐาน: บัญชีใหม่ (< 60 วัน) มีโอกาสเป็นม้าสูง

WHAT — ตัวชี้วัดอะไรบอกได้ชัดที่สุด?
  → ใช้คอลัมน์: Fan_In_Count, Outflow_Ratio_Pct, Avg_Time_To_Outflow_Min
  → สมมติฐาน: Fan-in > 4 + Time-to-Outflow < 15 นาที = ม้า

WHERE — ช่องทางไหนเสี่ยงที่สุด?
  → ใช้คอลัมน์: Channel
  → สมมติฐาน: PromptPay = ช่องทางหลักของบัญชีม้า

WHEN — เกิดช่วงเวลาไหน?
  → ใช้คอลัมน์: Timestamp
  → สมมติฐาน: บัญชีม้าทำธุรกรรมกลางคืน (22:00-06:00)

WHY — ทำไมโอนออกทันที?
  → ใช้คอลัมน์: Avg_Time_To_Outflow_Min, Min_Balance
  → สมมติฐาน: Hit & Run = ตัดร่องรอยก่อนถูกตรวจจับ

HOW — ใช้วิธีไหนตรวจ?
  → คำนวณ Risk Score รวม 4 Signal
  → กำหนดเกณฑ์ Score ≥ 70 = High Risk

SLIDE 5 — HYPOTHESIS: FAN-IN/FAN-OUT + LAYERING
================================================================

หัวข้อ: "Fan-in / Fan-out Pattern คืออะไร?"

[Diagram กลางหน้า]

    เหยื่อ 1 ──┐
    เหยื่อ 2 ──┤──→ Layer 1 ──→ Layer 2 ──┐
    เหยื่อ 3 ──┤   (บัญชีม้า)   (บัญชีม้า) ├──→ Layer 3+ ──→ Cash Out
    เหยื่อ 4 ──┘                            ┘   (บัญชีม้า)

         ←── Fan-in ──→  ←──── Fan-out / Layering ────→

3 Hypothesis หลัก:

H1: บัญชีม้ามี Fan-in Count สูงกว่าบัญชีปกติ
    → วัดด้วย: COUNT(DISTINCT From_Account) per To_Account

H2: บัญชีม้าโอนออกภายใน 15 นาทีหลังรับเงิน (Hit & Run)
    → วัดด้วย: Avg_Time_To_Outflow_Min

H3: ระบบเดิมจับ Layering ได้น้อยลงตามชั้นที่ลึกขึ้น
    → วัดด้วย: Detection_Rate_Pct ลดลงทีละ ~35%

SLIDE 7 — DATA QUALITY CHECK (Rubric Sec 3: ห้ามลบ Outlier!)
================================================================

หัวข้อ: "การตรวจสอบคุณภาพข้อมูล"

[ Checks ที่ทำใน Notebook ]

1. Missing Values
   → ผลลัพธ์: ไม่ลบรายการ
   → เหตุผล: ค่า Null อาจเป็น signal เอง

2. Outlier Analysis (IQR Method)
   → Boxplot ของ Amount
   → ผลลัพธ์: Outlier ส่วนใหญ่ = บัญชีม้า
   → เหตุผล: ห้ามลบ — นี่คือ Fraud Signal!

3. Class Imbalance
   → Pie Chart: 94.7% Normal vs 5.3% Mule
   → ใช้ Log Scale ในกราฟอื่นเพื่อให้เห็น Mule

4. Self-Transfer Check
   → From_Account = To_Account → ลบได้

Key Message ตาม Rubric:
"ในการตรวจจับ Fraud — ค่าผิดปกติคือสัญญาณ ไม่ใช่ noise"

SLIDE 14 — RULE-BASED DETECTION (สำคัญ! Midterm พลาดข้อนี้)
================================================================

หัวข้อ: "ระบบ Risk Scoring แบบกฎเกณฑ์"

Rule 1 — Fan-in Alert
   IF Fan_In_Count > 4 THEN +30

Rule 2 — Hit & Run Alert
   IF Avg_Time_To_Outflow_Min ≤ 15 THEN +35

Rule 3 — Account Draining
   IF Outflow_Ratio_Pct > 95 THEN +25

Rule 4 — New Account Risk
   IF Account_Age_Days < 60 AND Fan_In_Count > 3 THEN +10

[ เกณฑ์ตัดสิน ]
   Score ≥ 70  → 🔴 High Risk     | Freeze ทันที
   Score 40-69 → 🟡 Watch          | Enhanced Monitoring
   Score < 40  → 🟢 Normal         | Pass Through

[ ผลลัพธ์ที่วัดได้ ]
   Recall (จับ Mule ได้)         : วัดจริงจาก Account_Summary
   False Positive Rate           : วัดจริงจาก Account_Summary
   Precision                     : คำนวณ TP / (TP + FP)

TABLEAU SETUP TIPS (ก่อน Export)
================================================================

ทุก Sheet:
  - ลาก Layer ลง Filters → ติ๊ก Victim ออก
  - คลิก Show Me → เลือก Box-and-Whisker (สำหรับ Sheet 1, 2)
  - ดับเบิลคลิก Title → ตรวจ format ให้สวย

Sheet 4 (Time to Outflow):
  - คลิกขวาที่แกน Y → Add Reference Line → 15
  - Label: "Hit & Run Threshold"

Sheet 5 (Layering Detection):
  - Sort Layer 1, 2, 3+ จากซ้ายไปขวา
  - ใส่ Label ค่า %

สีที่ Consistent ทั้ง Workbook:
  Layer 1   = #d62728 (แดงเข้ม)
  Layer 2   = #ff7f0e (ส้ม)
  Layer 3+  = #ffbb78 (ส้มอ่อน)
  Normal    = #1f77b4 (น้ำเงิน)
  Victim    = #7f7f7f (เทา)
